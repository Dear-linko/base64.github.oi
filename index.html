Base64 encoding instructions

Base64 is a representation method based on 64 printable characters to represent binary data. Since 2^6=64, every 6 bits is a unit, corresponding to a certain printable character。

Base64 is often used to represent, transmit, and store some binary data in situations where text data is usually processed, including MIME emails and some complex data in XML.

Base64 encoding requires converting three 8-bit bytes (3*8=24) into four 6-bit bytes (4*6=24), and then adding two 0s in front of the 6 bits to form an 8-bit word section form. If the remaining characters are less than 3 bytes, they will be filled with 0, and the output characters will use =, so 1 or 2 =s may appear at the end of the encoded output text.

In order to ensure that the output code bits are readable characters, Base64 has developed a code table for unified conversion. The size of the encoding table is 2^6=64, which is where the name Base64 comes from.

The printable characters in Base64 include letters A-Z, a-z, and numbers 0-9, so there are 62 characters in total, and the two printable symbols are different in different systems.

Here are the basic steps for Base64 encoding:

  • Divide the data into groups of 3 bytes (24 bits).
  • Convert each byte to 8-bit binary form.
  • Divide the 24-bit data into 6-bit groups to get 4 6-bit groups.
  • Convert each group of 6 bits to the corresponding Base64 character.
  • If the data is less than 3 bytes, fill it.
  • Concatenate all converted Base64 characters to form the final encoded result.

The process of decoding Base64 encoding is the opposite of encoding, converting each Base64 character into a corresponding 6-bit binary value, and then combining these 6-bit values into the original binary data.

Base64 encoding has the following characteristics:

  • The encoded data length is always about 1/3 longer than the original data.
  • The encoded data can contain any combination of A-Z, a-z, 0-9, and two additional characters.
  • Base64 encoding is a reversible encoding method that can restore the original data through decoding.
